1Attendance Eligibility Engine

Problem
A college system stores student name, attendance percentage, and class status. Build a rule-based checker that prints whether the student can enter the exam hall and also whether they need a warning message.
Constraintss
  • Attendance is an integer from 0 to 100
  • If attendance is below 75, student is not eligible
  • If attendance is between 75 and 79, show "Eligible with Warning"
  • If attendance is 80 or above, show "Eligible"
  • Use variables and conditions only
Sample Input
Name: Kavya Attendance: 78
Expected Output
Student Name: Kavya Attendance: 78% Status: Eligible with Warning

2Dynamic Cart Analyzer

Problem
An online store keeps product prices in an array. Write a program to calculate total bill, highest item price, and final amount after discount rules.
Constraints
  • Array must contain at least 5 prices
  • If total is above 5000, apply 12% discount
  • If total is above 8000, apply 20% discount
  • Use array processing and conditions
  • Print total, max item price, discount, and final payable amount
Sample Input
[1200, 2300, 900, 1800, 700]
Expected Output
Total Amount: 6900 Highest Item Price: 2300 Discount: 828 Final Amount: 6072

3Inventory Priority Report

Problem
A warehouse stores products as objects with name, price, stock, and category. Show only low-stock items and sort them by price in descending order.
Constraints
  • Each product should have 4 fields
  • Low stock means stock less than 10
  • Use filter() and sort()
  • Sort by price from high to low
  • Output must show product name, category, and stock
Sample Input
[ { name: "Printer", price: 14000, stock: 4, category: "Office" }, { name: "Mouse", price: 800, stock: 25, category: "Accessories" }, { name: "Router", price: 2500, stock: 6, category: "Networking" } ]
Expected Output
Low Stock Items: Printer - Office - 4 Router - Networking - 6

4Library Penalty Calculator

Problem
Create a function that calculates fine based on overdue days and membership type. Premium users get a smaller fine than regular users.
Constraints
  • Overdue days must be 0 or more
  • Regular user fine:
    • 1 to 5 days: ₹10/day
    • 6 to 10 days: ₹20/day
    • above 10 days: ₹50/day
  • Premium user fine:
    • 1 to 5 days: ₹5/day
    • 6 to 10 days: ₹10/day
    • above 10 days: ₹25/day
  • Use a function and conditions
Sample Input
Days: 8 User Type: Premium
Expected Output
Late Days: 8 User Type: Premium Fine: 80

5Result Card Renderer

Problem
Store student marks in an object, calculate total, average, highest subject, and pass/fail status. Display the complete report in the webpage using DOM.
Constraints
  • Use exactly 3 subjects
  • Marks must be between 0 and 100
  • Pass only if all subjects are 40 or above
  • Show highest scoring subject
  • Use object, function, and DOM update
Sample Input
Name: Arjun Math: 91 Science: 74 English: 63
Expected Output
Student Name: Arjun Total: 228 Average: 76 Highest Subject: Math Result: Pass
